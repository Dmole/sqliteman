Find Dialog

This dialog allows you to search for rows in the current table. It is a modeless dialog and will remain visible until either you dismiss it or you open a non-table (a query or view) in the Data Viewer. If your window manager can find space on the screen, the Find Dialog will be displayed without overlapping the sqliteman main window.

The search conditions are specified similarly to the Query Builder but they are interpreted internally to sqliteman instead of being used to construct an SQL query and there is no column selection or ordering clause. You can search for the first matching row, the next matching row after the currently selected one, or all matching rows. The output from Find All looks a bit like the output from running a query, but it is actually the real table with all the non-matching rows hidden. Since it is the real table, you can edit the rows, which you cannot do with output from the Query Builder. Note however that making a Table Snapshot will snapshot the whole table, not just the non-hidden rows. This may change in some future version of sqliteman.

The More button adds another match condition to the search and the Less button deletes the last match condition. If you add several conditions you can choose whether the search finds rows which match all of them or match any of them.

The Find First button finds the first matching row in the table and selects the first field as if you had clicked on it. If there is no matching row, the Error Log will show "Not found".

The Find Next button finds the next matching row in the table after the row currently selected and selects the first field as if you had clicked on it. If there is no matching row, the Error Log will show "Not found".

The Find All button hides all rows in the table which do not match the search conditions, so that you can easily examine and edit those which do match. If there are no matching rows, the whole table is shown and the Error Log will show "No match found".

The Finished button dismisses the dialog. Any rows hidden after a Find All are made visible again. Closing the dialog box has the same effect.
